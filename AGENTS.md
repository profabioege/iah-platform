# Regras Gerais

- Nunca criar funcionalidades que não tenham sido solicitadas.
- Nunca modificar arquivos sem autorização.
- Sempre explicar o plano antes de implementar.
- Sempre perguntar quando houver ambiguidade.
- Manter o código simples e organizado.
- Priorizar legibilidade em vez de complexidade.
- Não adicionar dependências desnecessárias.
- Não criar telas fictícias apenas para demonstração.
- Não utilizar dados mockados sem autorização.
- Toda funcionalidade deve ser documentada.
- Toda decisão arquitetural relevante deve ser registrada.

Este documento deve ser seguido durante todo o desenvolvimento do projeto.
